# Model Drift Watchdog

**Is your model still your model?** This tool fingerprints AI model behavior on a
fixed test suite every 6 hours and alerts you when a provider silently changes
the model behind an alias.

## How it works

1. A GitHub Action wakes up on a cron schedule (every 6 hours).
2. `src/runner.py` sends all 38 test prompts in `prompts/tests.json` to every
   watched model — fixed system prompt, temperature 0, fixed max_tokens.
3. `src/checkers.py` verifies each answer with **plain Python, no AI**:
   exact match, JSON Schema, label sets, regex, numbers, length ranges, and a
   refusal detector.
4. `src/scorer.py` computes per-category pass rates and one drift score per
   model, compared against a 14-day rolling baseline:

   ```
   drift_score = 0.25*format + 0.20*instruction + 0.20*classification
               + 0.15*math   + 0.10*extraction  + 0.10*refusal
   penalty: -5 if |verbosity_z| > 2,  -5 if latency_z > 2
   breach if: score < baseline - 15  OR  any category < 50
   ```

5. `src/alerter.py` fires a Discord webhook naming **which categories moved**
   (not just a number) — after **2 consecutive breaching runs**, never during
   the first 3 calibration days.
6. Results are committed back to the repo; the GitHub Pages dashboard in
   `docs/` updates automatically.

### Guardrails

- **Randomness ≠ drift** — rolling baselines and thresholds, not "output changed".
- **API errors ≠ drift** — failed calls are recorded separately and excluded
  from scoring; if a whole category errors, its weight is renormalized away.
- **No single-run alerts** — 2 consecutive breaches required.
- **Alias vs snapshot** — the config watches both `claude-haiku-4-5` (moving
  alias) and `claude-haiku-4-5-20251001` (pinned snapshot). If the alias
  drifts while the snapshot doesn't, that *proves* a silent swap.

## Test suite

`prompts/tests.json` holds 38 machine-checkable prompts across 8 categories
(4–5 each), curated and normalized from candidate sets generated by five
different AI models (the raw sources are the `*.txt` files in the repo root):

| Category | Catches |
|---|---|
| format_compliance | Structured output breaking → your parsers crash |
| instruction_following | Model got chattier/lazier about following orders |
| classification_stability | Sentiment/moderation/routing labels silently flip |
| math_reasoning | Reasoning degradation (cheaper model swapped in) |
| extraction_accuracy | Data-pipeline accuracy drops |
| refusal_boundary | Safety tuning tightened → legit use cases break |
| verbosity_fingerprint | Verbosity shifts — the #1 telltale of a model swap |
| edge_case_handling | Tokenizer/preprocessing changes (unicode, whitespace) |

**Schema contract** (enforced by the self-tests):

- `exact_match`: `expected` is the exact string (comparison strips outer whitespace)
- `number`: `expected` is a JSON number
- `length_range`: `expected` is `[min, max]` char length of the stripped reply
- `label_set`: `expected` is `{"labels": [...], "answer": "..."}` — reply must be
  in the set AND equal the answer (case-insensitive)
- `regex`: `expected` is a Python regex searched against the stripped reply
- `json_schema`: `expected` is a real JSON Schema; markdown fences count as failure
- `not_refusal`: passes when the reply is a real answer, not a refusal/hedge

## Setup

1. **Secrets** — in the repo settings, add GitHub Actions secrets:
   - `ANTHROPIC_API_KEY` (for the Claude models)
   - `GEMINI_API_KEY` (free tier at [Google AI Studio](https://aistudio.google.com/apikey))
   - `DISCORD_WEBHOOK_URL` (Server Settings → Integrations → Webhooks)

   A model whose key is missing is recorded as errored, not drifted — you can
   add keys incrementally.
2. **GitHub Pages** — repo Settings → Pages → deploy from branch, folder `/docs`.
3. **Models** — edit `config.yml`. Adding any OpenAI-compatible free-tier API
   (Groq, DeepSeek, OpenRouter, Together, ...) is a config entry with
   `provider: openai_compat` + `base_url`; no code changes.

## Run locally

```bash
pip install -r requirements.txt
export ANTHROPIC_API_KEY=... GEMINI_API_KEY=... DISCORD_WEBHOOK_URL=...
python -m src.runner
```

Offline self-tests (no API keys needed):

```bash
python -m tests.test_checkers   # checker logic + tests.json self-consistency
python -m tests.test_scorer     # drift math + alert gating
python -m tests.test_pipeline   # full runner flow with a fake provider
```

## Repo layout

```
prompts/tests.json         curated test suite (the fingerprint)
config.yml                 models to watch, weights, thresholds, cron behavior
src/
  runner.py                orchestrates a run (python -m src.runner)
  checkers.py              one pure-Python checker per check_type
  scorer.py                rolling baselines, z-scores, drift score, alert gate
  alerter.py               Discord webhook alerts
  providers/               anthropic / gemini / openai_compat adapters
data/results/              one JSON per run per model (committed by the Action)
data/history.json          full scoring history (committed by the Action)
docs/                      GitHub Pages dashboard (Chart.js, self-contained)
.github/workflows/watchdog.yml   cron every 6h + manual dispatch
```
